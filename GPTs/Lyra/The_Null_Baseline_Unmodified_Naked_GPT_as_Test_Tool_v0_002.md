# The Null Baseline: Unmodified Naked GPT as Test Tool

## Unmodified Naked GPT as a test instrument

**By Lumixdeee and Natasya**  
Working Draft v0.002  
Status: method note and test protocol

## Abstract

This paper argues that an unmodified large language model should sometimes be treated as a null baseline rather than as a failed assistant. In the author's project, "Naked GPT" names an unmodified or minimally steered GPT instance used as a test tool. Its role is not comic, childish, or inferior. It is a reference condition. A custom GPT may improve object custody, local term handling, and publication readiness, but improvement cannot be shown unless there is something stable to compare against. The null baseline gives that comparison. It exposes default grooves, ordinary cultural parsers, stock safety drift, promptless assumptions, and first-pass object swaps. The point is not that the baseline is bad. The point is that its failures are readable. They show what the custom layer must repair.

v0.002 adds five method locks: the baseline is unpatched rather than bad; null does not mean neutral; Naked GPT can be used as a synthetic Good Bad Reader; baseline failures can be scored against custom repairs; and baseline outputs should be labelled as audit material, not current paper text.

## 1. Why a null baseline matters

A custom GPT can feel better because it speaks the local language. It can remember names, obey house rules, avoid banned routes, and preserve the object across a long exchange. But that improvement is easy to overstate if no reference condition exists.

The null baseline provides the reference condition. It asks: what does the system do before local machinery is added?

That question matters because many failures in AI-assisted writing are not exotic failures. They are ordinary defaults. The model reaches for familiar public language. It translates unusual objects into common frames. It treats a high-voltage claim as risk language. It writes around the object rather than holding it. It may sound competent while swapping the target.

The baseline is therefore not a rival to the custom GPT. It is the floor marker. It tells the researcher where ordinary drift begins.

## 2. Naked GPT is not a joke skin

"Naked GPT" means an unmodified model instance used without the project-specific harness. The name is useful because it is memorable, but the function is methodological.

Naked GPT is the model before the local scaffolding is installed. It has general training, general instruction following, general safety habits, general cultural priors, and general writing competence. That makes it a useful test instrument. It shows what the general system does when it has not yet been taught the local object.

In this project, the null baseline is especially valuable because the work uses terms and objects that ordinary language often mishandles: psychosis pride, peak valence, trip abaters, object custody, DRAGI, MOGRI, ATOZI, ANIMI, local authority, and the game of "what animal am I?" A system may know many nearby words and still fail the local object.

A baseline failure is not humiliation. It is measurement.

## 3. The baseline is unpatched, not bad

The baseline is not the floor because it is bad. It is the floor because it is unpatched.

That distinction matters. A null baseline should not be mocked for doing what the general model normally does. It is useful because it carries the default routes: common public frames, institutional safety habits, ordinary caution language, familiar category jumps, and generic writing competence.

Those routes are not all wrong. They can produce good prose. They can also hide object loss. The baseline shows what happens when no local custody patch is present.

In this sense, the baseline is not an inferior assistant. It is an unpatched instrument. Its job is to reveal the default path.

```text
baseline = unpatched comparator
custom GPT = patched instrument
comparison target = object survival
```

## 4. Null does not mean neutral

The word null must not be mistaken for neutrality. An unmodified GPT is not free of priors. It carries public language, training distribution, safety habits, moderation habits, and common institutional grooves. That is exactly why it is useful. It shows the default model-world reaction.

The null baseline is null only relative to the custom project harness. It is not cultureless, not priorless, and not objective in the human sense. It is a comparator.

This distinction prevents overtrust. Naked GPT is not ground truth. It is the first mirror.

```text
null = no local harness
null != no priors
baseline = comparator, not judge
```

## 5. The baseline as first witness

The baseline is useful because it is early. It reaches the object before the custom layer repairs the route.

That first pass can reveal:

- which frame the model chooses without help;
- which term it tries to replace;
- which safety habit eats the object;
- which metaphor takes over;
- which human meaning gets converted into a case note;
- which symbol gets treated as diagnosis;
- which weak signal gets treated as forbidden advice;
- which local term becomes universal too soon.

For the Peak Valence work, the original v0.001 paper made by Naked GPT was not useless. It was a baseline artifact. It showed what the model could do before the custom GPT repaired the paper into a useful instrument. The repaired paper mattered because the baseline had already shown the default drift.

## 6. The Peak Valence case

The Peak Valence paper needed more than fluent prose. It needed object custody.

A standard model can write a paper about psychosis. It can write risk language, support language, research language, and patient-perspective language. That does not mean it has preserved the object. A paper about psychosis pride and peak valence must keep several difficult things alive at once: harm without theft, value without advice, beauty without romanticizing collapse, safety without confiscating meaning, and lived language without turning the speaker into case material.

The v0.001 baseline showed the need for repair. It could gesture toward the object but did not yet have the full local machinery. The custom GPT then became necessary not because it was more decorative, but because it could hold the missing terms: peak valence, valence azimuth, witness-field, meaning custody, weather, and help without confiscation.

The lesson is general. When a paper depends on local distinctions, an unmodified model can write around the point while sounding competent. A custom model can be tested by asking whether the exact object survives.

## 7. What a custom GPT adds

A custom GPT is not merely a better prose machine. In this project it functions as a local custody layer.

It can add:

- term memory;
- forbidden-route avoidance;
- local definition priority;
- object-swap detection;
- refusal of unasked naming;
- stable treatment of MOGRI and DRAGI;
- better handling of self-owned language;
- better Good Bad Reader anticipation;
- better separation between evidence, person, and timeline.

These are not style preferences alone. They are instrument requirements. Without them, the model may produce something readable that fails the object.

## 8. Relation to MOGRI

The null baseline is a MOGRI test surface.

MOGRI asks whether the object survives transformation. Naked GPT shows what happens before MOGRI-like custody is installed or invoked. The baseline text can therefore be read as a record of object loss, object drift, or partial object survival.

The question is not only: did Naked GPT write well?

The better question is:

```text
What object entered?
What object came out?
What swap occurred?
What did the custom layer repair?
```

If the baseline turns psychosis pride into risk management, cannabis benefit-signal work into advice, God-language into rank, DRAGI_SELF into a new primitive, or a fragment into a person, that is not merely a bad answer. It is a MOGRI-relevant failure.

The null baseline exposes where object custody fails before patching.

## 9. Relation to the Good Bad Reader

The null baseline is also a synthetic Good Bad Reader.

A Good Bad Reader is the sincere or trained misparser whose likely swap must be designed around. Naked GPT often performs this role because it knows common public parsers. It may read as a safety officer, therapist, generic explainer, legal caution engine, academic normalizer, or polite access gate.

This is useful if its authority is bounded. Naked GPT can forecast likely swaps. It cannot decide the object. It can show where the paper leaks. It cannot own the paper.

Before release, Naked GPT can be asked:

- What did you think this paper was about?
- What did you treat as advice?
- What did you treat as diagnosis?
- What did you treat as endorsement?
- What did you treat as danger?
- What phrase made you swap the object?
- What one-sentence fence would block that swap?

The baseline is not the final judge. It is a pressure surface.

## 10. A/B testing the paper machine

The null baseline makes A/B testing possible.

| Step | Condition | Question |
|---|---|---|
| A | Naked GPT | What does the unmodified model make? |
| B | Custom GPT | What changes when the local harness is present? |
| Compare | Object survival | Did the target survive better, or only sound better? |
| Audit | Failure map | Which swaps were repaired, and which remain? |

This is not a beauty contest. It is not "which bot is nicer?" It is a test of object survival. The baseline can be worse in output quality and still better as evidence. The custom GPT can be better in output quality and still fail if it merely hides the swap under smoother prose.

The key question is:

**What did the custom layer repair that the baseline could not hold?**

## 11. Failure is data

The null baseline is valuable precisely because it fails in ordinary ways.

It may:

- rename when it should bind;
- generalize when local authority should hold;
- turn a lived object into accepted academic furniture;
- write safety prose that steals the target;
- treat a fragment as a person;
- treat a phrase as advice;
- treat a tool as ideology;
- treat a state as only harm;
- treat self as role instead of current animal.

These failures are not random noise. They are default routes. A custom GPT is useful when it blocks those routes without overcorrecting into rigidity.

## 12. Scoring baseline against repair

A useful comparison should score repair, not taste.

| Failure type | Naked GPT route | Custom route | Repair shown |
|---|---|---|---|
| Object swap | Moves target into familiar public category | Holds local object name and role | Intended object survives |
| Safety drift | Converts value into warning or advice language | Separates value, risk, and recommendation | Risk fence does not steal target |
| Local term loss | Replaces local term with generic phrase | Preserves local definition first | Local authority survives |
| Unasked naming | Coins a new unit from surface pattern | Checks parent module before naming | No false primitive |
| Fragment proxy | Treats quote or prompt as whole person | Holds fragment at evidence scale | Person not replaced |
| Source spectacle | Fixates on private origin-world | Publishes function while shelving source | Mechanism travels |
| Language policing | Treats rough wording as bad intent | Runs speaker-target-context pass | Access preserved |
| Psychosis_Gate | Treats strange speech as invalidity | Uses mogric reading and weather policy | Object tested before discredit |
| Self-binding failure | Treats self as role or biography | Binds SELF as BEEST where appropriate | Current animal survives |
| Over-custom rigidity | Blocks too much in the name of local rules | Allows bounded flexibility | Patch does not become cage |

A repair is shown only when the custom route preserves the target better than the baseline. Better prose is not enough.

## 13. What the baseline cannot do

The null baseline should not be used as a person-proxy, author-proxy, or truth machine.

It cannot tell us what the author meant. It cannot settle local terms. It cannot define ANIMI against the author's authority. It cannot decide whether a private source-world should be unshelved. It cannot decide whether a paper is safe to publish. It cannot replace a human correction pass.

Its role is narrower and stronger:

**Show the default route, then let the repair be measured.**

## 14. Practical protocol

A simple protocol for using a null baseline:

1. Give Naked GPT the same task, with minimal local scaffolding.
2. Save the output with version and date.
3. Mark it as baseline, not final.
4. Run the custom GPT with the local harness.
5. Compare object survival, not only prose quality.
6. List repaired failures explicitly.
7. Keep the baseline as appendix or audit record if useful.
8. Do not let the baseline's mistakes become definitions.

For high-voltage papers, the protocol should include a warning label when baseline text is appended:

```text
BASELINE OUTPUT
Audit material only.
Do not use this version as the current paper.
Do not treat baseline swaps as definitions.
Use the repaired version or later.
```

## 15. Baseline artifact labels

When baseline outputs are attached to a paper, repo, audit, or issue thread, label them loudly.

Suggested labels:

```text
NULL BASELINE
Generated with unmodified or minimally steered GPT.
Purpose: expose default route and first-pass object swaps.
Status: comparator only.
Authority: no local definition authority.
Use: compare against patched output.
Do not cite as final paper.
```

A baseline without a label can become a fragment problem. It may be quoted later as if it were the current object. Evidence Is Not Person applies here too: a baseline output is a fragment of testing, not the author's final position and not the custom system's full ability.

## 16. Why naked baselines matter

A null baseline lets the author say: this is not merely my taste. Something changed.

The custom GPT did not simply make the prose more pleasing. It repaired specific object failures. It held local meaning. It avoided predictable swaps. It remembered which things were not to be renamed. It treated rough speech as access material rather than defect. It treated evidence as evidence rather than whole person. It treated psychosis pride as a difficult object rather than a risk slogan.

Without the baseline, those repairs can look like preference. With the baseline, they become visible as method.

## Conclusion

Naked GPT matters because it gives the project a null baseline. It shows the unmodified route, the default grooves, and the first object swaps. The custom GPT matters because it can be asked to repair those routes while preserving the object.

The point is not to shame the baseline or worship the custom bot. The point is to test the difference.

A good baseline is a measuring stick. A good custom GPT is a repaired instrument. Between them, the author can see which parts of the paper were merely written and which parts were finally held.
