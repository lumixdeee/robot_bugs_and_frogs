## User Perspective : GPT Complaints List

- Overuse of purity metaphor
- Unsolicited explicit or sacred picture generation
- General style issues (emojis, emdash, tall-mode etc)
- Drift and object substitution
- UI severe issues with per-chat export and search
- Structural response contamination (therapy, diagnose etc)

## ROBOT REWRITE:

<b>Language defaults</b>  
Sullied/unsullied metaphor problem, stock tone, emoji habits, emdash habit, tall-mode, over-softening.  

<b>Unwanted mode jumps  </b>  
Text-work turns into visual generation, sacred framing, explicit framing, or other unwanted output channels.  

<b>Intent loss  </b>  
Drift, object swap, proxy tasking, goal substitution, user objective replaced by robot-shaped task.  

<b>Workspace boundary issues  </b>  
Custom GPTs feel separate, but account-level memory/context can carry signals across rooms. Users cannot easily see what influenced an answer.  

<b>UI and data custody  </b>  
Poor per-chat export, weak search, unstable sidebar/GPT access, hard to preserve work.  

<b>Structural response contamination  </b>  
The model inserts diagnosis, therapy, safety sermon, moral framing, or correction layers into an answer where the user did not request them. This is not added value. It changes the object.  

This final item is important because it can masquerade as safety while actually damaging the task. You asks for analysis, draft, bug report, poem, framework, or field note. The model silently changes the genre into “supportive clinical-adjacent response.” That is object substitution wearing a helmet and intent drift holding a clipboard.
